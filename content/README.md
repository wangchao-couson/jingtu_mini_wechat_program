# 京途内容配置说明

这个文件夹是后续日常更新的主要入口。尽量只改这里的文字、图片路径和排序，不改 `pages/` 里的页面代码。

## 文件结构

- `site.js`
  - 全站基础文案、首页/发现页主视觉、审核友好的提示文案。
  - 适合改品牌 slogan、顶部大图、咨询说明。
- `teams.js`
  - 队伍/活动专题配置。
  - 适合新增队伍、修改日期地点、替换封面、设置领队微信、调整标签和行程。
- `roles.js`
  - 领队、摄影师、志愿者等个人主页配置。
  - 适合更新个人介绍、作品图、资历数据、微信号。
- `images/`
  - 预留给后续新增图片。
  - 当前第一版先复用 `/assets/images/trips/` 下的实景图片。

## 更新队伍

打开 `teams.js`，复制一个队伍对象，修改：

- `id`：保持唯一，例如 `team-haituo-camp`。
- `status`：`registering` 招募中，`upcoming` 即将开始，`finished` 已结束。
- `cover`：封面图路径，建议使用实景图。
- `leaderId`：对应 `roles.js` 里的角色 id。
- `wechat`：咨询微信，页面只展示“咨询”，不做报名入口。

## 更新角色

打开 `roles.js`，复制一个角色对象，修改：

- `id`：保持唯一，例如 `leader-ajie`。
- `roleType`：`leader`、`photographer`、`volunteer`。
- `wechat`：用于咨询或沟通。
- `gallery`：作品或带队照片，建议全部使用真实户外图片。

## 图片建议

- 正式上线前，优先使用真实活动、真实路线、真实人物或背影照片。
- 避免使用过度营销、价格促销、明显广告感图片。
- 图片路径示例：`/assets/images/trips/preview-04.jpg`。
- 后续如果把图片放到本文件夹，可使用：`/content/images/xxx.jpg`。

## 审核友好原则

- 不写“立即报名”“支付定金”“限时抢购”。
- 使用“咨询领队”“添加微信了解行程”“由领队统一沟通安排”。
- 页面重点展示队伍介绍、领队/摄影师/志愿者角色、真实照片、路线信息和安全提示。
